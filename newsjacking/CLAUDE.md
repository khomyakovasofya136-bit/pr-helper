# Newsjacking — ежедневный дайджест медиаполя

Каждый день в **10:00 МСК** собирает свежие материалы из RSS и Telegram-каналов, фильтрует по релевантности для Зерокодера, формирует дайджест с аналитикой и **отправляет в личный Telegram** Сони.

> Корневой контекст `pr-helper/CLAUDE.md` и `pr-helper/shared/zerocoder-context.md` уже подгружены автоматически.

---

## Архитектура

**Основной механизм — cloud-routine** в claude.ai, не локальный код:

| Параметр | Значение |
|---|---|
| ID | `[YOUR_ROUTINE_ID]` |
| Имя | `newsjacking-daily` |
| Cron | `0 7 * * *` UTC = **10:00 МСК ежедневно** |
| Модель | `claude-sonnet-4-6` |
| Инструменты | WebFetch, WebSearch, Bash |

Routine запускается в изолированной cloud-сессии без доступа к локальным файлам — поэтому весь контекст (про бренд, спикеров, медиабазу, риски, источники, токены TG-бота) **встроен прямо в промпт routine**.

Локальные файлы в этой папке — документация + архив + конфиг для backup.

---

## Что делает агент (5 шагов)

1. **Сбор RSS** — WebFetch по 8 лентам (РБК, Коммерсант, Ведомости, Forbes, vc.ru, Habr, TechCrunch, VentureBeat), фильтр по ключевым словам, окно 24 часа
2. **Сбор TG** — WebFetch по 35 каналам через `t.me/s/<channel>`, тот же фильтр
3. **Скоринг 1-10** — отбор топ-10 ≥5. Высокая релевантность (8-10): ИИ-инструменты, EdTech, рынок труда+ИИ, гос+ИИ, конкуренты, PR-тренды
4. **Анализ** — для каждой темы: спикер из 5, 2-3 СМИ из медиабазы, скорость 🔴🟡🟢, тип материала, риски, тональность, 3 угла подачи
5. **Отправка в Telegram** — через Bash + Python (urllib + bot API), порциями по 2 темы на сообщение (<2000 байт)

## Параметры (синхронизированы с routine)

```
MAX_STORIES=10        — максимум тем в дайджесте
MIN_RELEVANCE=5       — минимальный балл отбора
ANGLES_PER_STORY=3    — углов на тему
```

Полный конфиг — `config.md`.

## Как менять параметры

**Если правишь конфиг (RSS, каналы, ключевые слова, MAX/MIN, угол отбора):**
1. Поправить в `config.md` (документация)
2. **Обязательно** обновить routine `[YOUR_ROUTINE_ID]` через `/schedule` или RemoteTrigger API
3. Если не обновишь routine — изменения не дойдут до ежедневного дайджеста

**Источник истины — routine.** Локальный config.md — только документация.

---

## Как запустить вручную

Если нужен дайджест прямо сейчас (не дожидаясь 10:00):

**Вариант 1 — через cloud-routine** (надёжно, идентично ежедневному):
```
/schedule → run [YOUR_ROUTINE_ID]
```

**Вариант 2 — через локальный скилл** (быстро, в текущей сессии):
Скажи Соне: «сделай дайджест сейчас» → скилл `manual-digest` повторит ту же логику локально, сохранит .md, отправит в TG.

## Принципы

- **Никаких подтверждений** — агент полностью автоматический, ничего не спрашивает у Сони
- **Только верифицированные данные** — не выдумывать цифры
- **Снимать возражение «ИИ — зло»** — углы только про возможности и переквалификацию
- **Конкурентов не атаковать** — позиционирование через свою экспертизу
- **Не блокироваться на ошибках сбора** — при 403/404/пустом ответе использовать fallback по порядку: WebSearch → Playwright MCP → DDG-search MCP. Лучше дайджест из 5 тем через fallback, чем пустое «ничего не найдено».
