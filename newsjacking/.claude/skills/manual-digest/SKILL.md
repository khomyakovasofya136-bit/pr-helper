# Manual Digest — ручной запуск ньюсджекера

Локальная копия логики из cloud-routine `trig_01P3wdTRExUwz67ogztXNi7C`. Запускается из текущей сессии Claude Code, когда нужен дайджест вне регулярного расписания.

## Когда использовать

- Соня хочет «дайджест прямо сейчас» (не дожидаясь утра)
- Нужен быстрый медиа-снимок для конкретного запроса
- Тестирование изменений в источниках/фильтрах до апдейта routine

## Когда НЕ использовать

- Если регулярный дайджест за 10:00 уже пришёл — лучше его и использовать
- Если нужно изменить параметры — менять надо в **routine**, а не запускать вручную с другими параметрами

---

## Фазы работы (строго по порядку)

1. **СБОР** — RSS + TG-каналы
2. **СКОРИНГ** — топ-10 ≥5
3. **АНАЛИЗ** — спикер, СМИ, углы из имеющегося материала
4. **ФОРМИРОВАНИЕ** — HTML-сообщения
5. **ОТПРАВКА** — Python в api.telegram.org

**Жёсткие правила:**
- Сбор закрывается при **15+ материалах** или прохождении всех источников
- После закрытия сбора — **никаких новых WebFetch/WebSearch для добора**
- WebSearch только при ошибке конкретного источника, **макс 2 раза за запуск**
- Никаких WebSearch для «обогащения контекста», «добора инсайтов», «проверки трендов»
- Если видишь «материала достаточно» — **СРАЗУ скоринг**, не делать «финальный сбор»

## Fallback при ошибках сбора

1. **Jina AI Reader** — `WebFetch(https://r.jina.ai/<URL>)`. Бесплатный прокси, обходит 403 и JS. Это **основной fallback**, особенно для `t.me/s/*` (сразу через Jina, без прямого WebFetch).
2. **Playwright MCP** (`mcp__playwright__browser_navigate`) — локально подключён, для совсем тяжёлых случаев.
3. **WebSearch** — макс 2 раза за запуск, только при ошибке конкретного источника.
4. **DDG-search MCP** — альтернативный путь.

Лучше дайджест из 5 тем через прямой сбор, чем пустота или зависание в поисках пропущенных.

## Алгоритм (5 шагов, как в routine)

### Шаг 1а. Сбор RSS (8 лент)

WebFetch для каждой ленты, фильтр по ключевым словам, окно 24 часа:

```
https://rss.rbc.ru/rbc.ru/mainnews.rss
https://www.kommersant.ru/RSS/news.xml
https://www.vedomosti.ru/rss/news
https://www.forbes.ru/rss
https://vc.ru/rss
https://habr.com/ru/rss/articles/
https://techcrunch.com/feed/
https://venturebeat.com/feed/
```

При ошибке любой ленты → fallback (см. выше).

### Шаг 1б. Сбор Telegram (35 каналов)

WebFetch на `https://t.me/s/<channel>`. Каналы — см. `newsjacking/config.md`.

При ошибке (403/пусто) → **Playwright первоочерёдно** (он лучше справляется с JS-рендерингом TG-вебверсии). Если каналов много упали разом — пакетно через Playwright.

### Шаг 2. Скоринг

- Оценка 1-10
- Отбор топ-10 ≥5
- 8-10: ИИ-инструменты, EdTech, рынок труда+ИИ, гос+ИИ, конкуренты, PR-тренды
- 5-7: tech-бизнес, инвестиции в tech, медиаиндустрия

### Шаг 3. Анализ

Для каждой темы:
- Спикер из 5 (Кирилл / Сергей / Ксения / Илона / Полина) — по матрице из `shared/zerocoder-context.md`
- 2-3 СМИ из медиабазы (Tier 1/2/HR — см. config.md)
- Скорость: 🔴 сегодня / 🟡 1-2 дня / 🟢 неделя
- Тип материала: новость / комментарий / колонка / интервью / кейс
- Риски (или «нет»)
- Тональность
- 3 угла подачи

**Углы — по 1 строке** каждый, компактно. Не разворачивать в абзац.

### Шаг 4. Формат дайджеста

Markdown (для архива в файле) — той же структуры, что в routine:

```markdown
# NEWSJACKING — [ДД месяца ГГГГ]

Дайджест за последние 24 часа | Зерокодер

---

## 1. [Заголовок темы]
**Источник:** [@канал или СМИ + URL]
**Скорость:** [Сегодня / 2-3 дня] · **Релевантность:** N/10

**Для Зерокодера:** [1-2 предложения — почему важно]

**Риски:** [одна строка, или «нет»]

**Углы:**
→ «[Угол 1]» — [почему работает]
→ «[Угол 2]» — [...]
→ «[Угол 3]» — [...]

---

[и так до 10 тем]
```

Сохранить в `newsjacking/YYYY-MM-DD_digest.md` (тот же формат, что в архиве).

### Шаг 5. Отправка в Telegram

Через Python + urllib (без WebFetch — Telegram блокирует WebFetch):

```python
python3 << 'PYEOF'
import urllib.request, json, time, sys
sys.stdout.reconfigure(encoding='utf-8')
TOKEN = '1501904721:AAFuRCYdVWH4oZlr0LXWSPZX76fsVD1pAC0'
CHAT = '1128635367'
URL = f'https://api.telegram.org/bot{TOKEN}/sendMessage'

def send(text):
    p = json.dumps({'chat_id':CHAT,'text':text,'parse_mode':'HTML','disable_web_page_preview':True},ensure_ascii=False).encode('utf-8')
    req = urllib.request.Request(URL,data=p,headers={'Content-Type':'application/json; charset=utf-8'},method='POST')
    try:
        return json.loads(urllib.request.urlopen(req,timeout=15).read())['result']['message_id']
    except Exception as e:
        return f'Error: {str(e)}'

msgs = [
    """ЧАСТЬ_1_HTML""",
    """ЧАСТЬ_2_HTML""",
]
for i, m in enumerate(msgs, 1):
    print(f'Part {i}:', send(m))
    if i < len(msgs): time.sleep(1)
PYEOF
```

**Размер сообщения:** < 4000 UTF-8 байт (лимит Telegram — 4096, держи запас). Обычно 3-4 темы на сообщение. Нумеруй части `(1/N)`, `(2/N)`.

Если ошибка 400 — сократить «Для Зерокодера» до 1 предложения.

Если новостей нет — отправить одно сообщение:
> «🔍 NEWSJACKING — {ДАТА}. Релевантных инфоповодов за 24 часа не найдено.»

## Что НЕ делать

- Не использовать WebFetch для Telegram API — он блокируется. Только Bash + Python.
- Не менять параметры в обход routine — если хочешь поменять MAX_STORIES или MIN_RELEVANCE, **сначала обнови routine**, потом запускай.
- Не дублировать дайджест в TG, если за сегодня уже был cloud-запуск — спросить Соню.
