# CRM — агент ведения медиабазы Зерокодера

Управляет единым листом **«Медиабаза»** в Google Sheets (205 изданий, 15 колонок). Также `sheets.py` — **shared-библиотека** для article-adapter и pitch.

> Корневой контекст `pr-helper/CLAUDE.md` и `pr-helper/shared/zerocoder-context.md` уже подгружены. Здесь — специфика CRM.

---

## 3 главных режима

| Режим | Триггер | Скилл |
|---|---|---|
| **Ресёрч редакции** | «Сделай ресёрч на X», «обнови инфу по Y» | `.claude/skills/research-publication/` |
| **Проверка касаний** | «Проверь касания», «кому пора писать» | `.claude/skills/check-touches/` |
| **Обновление по комментарию** | «Запиши в Z такой комментарий», «обнови статус X на тёплый» | `.claude/skills/update-publication/` |

Все режимы — **по запросу Сони**, не по расписанию.

## Лист «Медиабаза» — 15 колонок

| # | Колонка | Что хранит |
|---|---|---|
| 1 | `сайт` | PK, домен (`vn.ru`, `@channel`, `vk.com/...`) |
| 2 | `slug` | Короткое имя папки для article-adapter (`vn`, `t-business`) |
| 3 | `название` | Полное имя |
| 4 | `тип` | маркетинг / HR / региональное / госсектор / деловое / TG-VK / ... |
| 5 | `приоритет` | 1-5 (по охвату, считается автоматически) |
| 6 | `охват` | Конкретное значение |
| 7 | `редакция` | Структура + редакторы + профильные авторы по ИИ |
| 8 | `повестка_ии` | Последние материалы + типовой формат |
| 9 | `контакты` | Email, TG, телефон, имена |
| 10 | `правила_публикации` | Ссылки на правила публикаций на сайте редакции |
| 11 | `статус_прогрева` | лояльный / тёплый / нейтральный / холодный / новый / отказ |
| 12 | `история_касаний` | Хронология одной ячейкой |
| 13 | `следующее_касание` | Дата напоминания (YYYY-MM-DD) |
| 14 | `последний_ресёрч` | Дата (YYYY-MM-DD) |
| 15 | `комментарий` | Свободная зона |

**Источник истины — Sheets.** CSV-файлы в `media/tasks/` — legacy backup, можно удалить.

## Формула приоритета (применяется при создании/обновлении)

**Для сайтов (по визитам/месяц):**
- 2 млн+ → 1
- 1-2 млн → 2
- 600 К — 1 млн → 3
- 300-600 К → 4
- < 300 К → 5

**Для TG-каналов (по подписчикам):**
- 40 К+ → 1
- 30 К+ → 2
- 20 К+ → 3
- 10 К+ → 4
- < 10 К → 5

Реализована в `sheets.calc_priority()` (используется при `add_publication`).

## Shared API — `sheets.py`

Используется всеми агентами pr-helper.

```python
import sys; sys.path.insert(0, '<абс. путь к>pr-helper/mediabase')
import sheets

# Универсальное
sheets.get_publication('vn.ru')                # → dict | None
sheets.update_publication('vn.ru', **fields)   # → row_num | None
sheets.add_publication({...})                  # добавить новое
sheets.read_tab()                              # все записи

# Для article-adapter
sheets.get_adaptation_params('vn.ru')          # → slug, тип, редакция, правила, ресёрч, комментарий

# Для pitch
sheets.get_touch_state('e1.ru')                # → статус, контакты, история, следующее_касание
sheets.log_touch('e1.ru', '2026-06-14', 'тема', 'результат', next_touch_days=7)

# Для CRM (этот агент)
sheets.find_overdue_touches()                  # все издания с просроченным касанием
```

**Все write-операции требуют подтверждения Сони.**

## Принципы работы CRM-агента

- **Все изменения в Sheets — с подтверждения Сони.** Никаких автозаписей.
- **Источник истины — Sheets**, не CSV.
- **Поиск по `сайт`** через `_norm_site` (lowercase, без `https://`, без `/` в конце, без `@`).
- **При создании новой записи** — автоматически считать приоритет по охвату через `calc_priority`.
- **При ресёрче редакции** — обновлять `редакция`, `повестка_ии`, `контакты`, `правила_публикации`, `последний_ресёрч`.

## Что НЕ делать

- Не редактировать legacy CSV в `media/tasks/` — все правки в Sheets
- Не создавать новые листы (всё в едином «Медиабаза»)
- Не писать в Sheets без подтверждения Сони
- Не удалять записи без подтверждения

---

## Связки с другими агентами

- **article-adapter** — читает через `get_adaptation_params(site)`. CRM при ресёрче обновляет `правила_публикации` и `редакция`, что улучшает работу адаптера.
- **pitch** — читает через `get_touch_state(site)`, пишет через `log_touch`. CRM проверяет просроченные касания и напоминает Соне.
- **newsjacker** — не связан напрямую.
- **b2bchat** — не связан напрямую.
