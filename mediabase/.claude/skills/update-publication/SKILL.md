# Обновление записи

Универсальный режим: Соня говорит «обнови X в Y» — агент находит запись, показывает что меняется, после подтверждения пишет в Sheets.

## Триггеры

- «Запиши в [издание] комментарий: [текст]»
- «Обнови статус [издание] на тёплый»
- «Добавь контакт [editor@x.ru] в [издание]»
- «Поменяй приоритет [издание] на 2»
- «Запиши, что мы отправили [тему] в [издание] сегодня»
- «Добавь новое издание [название] с сайтом [URL]»

## Алгоритм

### Шаг 1. Найти запись

```python
import sheets
pub = sheets.get_publication(site)
```

Если не нашлась:
- Спросить: «Издание [X] не найдено в Медиабазе. Создать новую запись?»
- Если да — собрать минимум данных (сайт, название, тип, охват → автоматически приоритет)

### Шаг 2. Определить какие поля менять

Распарсить запрос Сони. Примеры:

| Запрос | Изменение |
|---|---|
| «запиши комментарий: «Х»» | `комментарий` ← новый текст (или + к существующему) |
| «обнови статус на тёплый» | `статус_прогрева` ← `тёплый` |
| «добавь контакт ivan@x.ru» | `контакты` ← добавить в существующее значение |
| «поменяй приоритет на 2» | `приоритет` ← `2` |
| «мы отправили "тему X" сегодня» | `log_touch(site, today, 'тема X', '', next_touch_days=7)` |
| «добавь новое издание X» | `add_publication({...})` с автоматическим приоритетом |

### Шаг 3. Показать diff

```
Изменения в [издание]:
  • статус_прогрева: [текущий] → [новый]
  • контакты: [текущий + новый]
  • комментарий: добавится «...»

Применить? (да / правки / отмена)
```

### Шаг 4. Записать с подтверждения

```python
sheets.update_publication(site, **changes)
```

Или, если новое издание:
```python
sheets.add_publication({
    'сайт': '...',
    'slug': '...',  # короткое имя
    'название': '...',
    'тип': '...',
    'приоритет': sheets.calc_priority(reach_int, тип),  # автоматически
    'охват': '...',
    # остальные поля пустые — заполнятся через research-publication
})
```

## Правила работы с конкретными полями

### `приоритет`
- Считается автоматически по `охват` через `sheets.calc_priority()`
- Менять вручную можно, но только с явным указанием Сони

### `slug`
- Только латиница без пробелов (`vn`, `t-business`, `74ru`)
- Если меняется — нужно переименовать соответствующую папку в `article-adapter/publications/`
- **Предупредить Соню** перед изменением slug

### `комментарий`
- Свободная зона. **Добавлять**, не перезаписывать (старые заметки могут быть важны)
- Формат добавления: `\n\n[YYYY-MM-DD] новая заметка`

### `история_касаний`
- **Не редактировать вручную** — использовать `sheets.log_touch()`. Это автоматически форматирует запись и обновляет `следующее_касание`.

### `правила_публикации`
- Только ссылки + описание в скобках, формат:
  ```
  https://X/rules (описание правил)
  https://X/native (описание нативного формата)
  ```

## Что обязательно прочитать

- `mediabase/sheets.py` — `update_publication`, `add_publication`, `log_touch`, `calc_priority`
- `mediabase/CLAUDE.md` — список и формат всех 15 колонок

## Что НЕ делать

- Не писать без подтверждения Сони
- Не перезаписывать `комментарий` или `история_касаний` — добавлять
- Не считать приоритет вручную — использовать `calc_priority`
- Не менять `slug` без переименования папки `publications/<slug>/`
- Не создавать дубликаты — сначала `get_publication` для проверки
