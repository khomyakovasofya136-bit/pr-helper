# Ресёрч редакции

## Триггеры

- «Сделай ресёрч на VN»
- «Обнови инфу по 74.ru»
- «Найди контакты редакции Х»
- «Посмотри что у Y с ИИ-материалами»
- «Что в редакции Z сейчас происходит»

## Что собирается

1. **Структура редакции** — разделы, отделы
2. **Редакторы по разделам** — имена, специализация
3. **Профильные авторы по ИИ** — кто пишет про ИИ, какие темы
4. **Последние 5-10 материалов по ИИ** — заголовок + дата + автор + URL
5. **Контакты** — email редакции, email редактора раздела, TG, телефон
6. **Правила публикации** — ссылка(и) на правила на сайте издания с описанием в скобках

## Алгоритм

### Шаг 1. Проверить текущую запись

```python
import sheets
pub = sheets.get_publication('vn.ru')
```

Если запись есть и `последний_ресёрч` свежий (< 30 дней) — спросить Соню: «По X ресёрч был [дата]. Делать новый или использовать сохранённый?»

### Шаг 2. Ресёрч источников

Через WebFetch / WebSearch:
- Сайт издания (главная, раздел «о редакции», раздел «контакты», раздел «правила публикации»)
- Поиск «<издание> редактор раздела технологии»
- Поиск «<издание> авторы AI 2026»
- Топ-10 свежих материалов по ИИ (по дате, по релевантности)

При блокировках — fallback через **Jina Reader** (`https://r.jina.ai/<URL>`).

### Шаг 3. Сформировать summary

В **компактном текстовом виде** (Соня хочет 10-15 колонок без дробления):

**Редакция (одна ячейка):**
```
Раздел «Технологии» — Иван Петров (ivan@vn.ru), профильно про ИИ с 2024.
Раздел «Бизнес» — Анна Сидорова.
Главред — Мария Иванова. Зам — Сергей Петров (s.petrov@vn.ru).
Отдел рекламы — Юлия (yulia@vn.ru, +7 999 123-45-67).
```

**Повестка ИИ (одна ячейка):**
```
3-5 материалов в неделю про ИИ. Формат: лонгриды с цифрами HH-индекса + кейсы.
Последние:
- «ИИ заберёт работу» (15.06.2026, Капустин) — https://...
- «Профессии под угрозой» (10.06.2026, Сидорова) — https://...
- «Кадровый дефицит» (05.06.2026, Иванов) — https://...
```

**Правила публикации (одна ячейка):**
```
https://vn.ru/help/native (правила нативных материалов)
https://vn.ru/blogs/rules (правила корпоративных блогов)
```

### Шаг 4. Показать Соне и подтвердить

```
Подготовил ресёрч по [издание]:

— Редакция:
  [полный текст]

— Повестка по ИИ:
  [полный текст]

— Контакты:
  [полный текст]

— Правила публикации:
  [ссылки]

Записать в Медиабазу? (да / правки / отмена)
```

### Шаг 5. Записать с подтверждения

```python
sheets.update_publication(
    'vn.ru',
    редакция='...',
    повестка_ии='...',
    контакты='...',
    правила_публикации='...',
    последний_ресёрч=date.today().isoformat(),
)
```

## Что обязательно прочитать перед стартом

- `mediabase/CLAUDE.md` — структура листа, формат полей
- `mediabase/sheets.py` — API библиотеки
- `pr-helper/shared/zerocoder-context.md` — спикеры, риски (для оценки уместности)

## Что НЕ делать

- Не записывать в Sheets без подтверждения Сони
- Не использовать выдуманные контакты или цитаты — только верифицированные
- Не делать ресёрч в обход листа (записать локально и забыть)
- Не дублировать структурные поля в `комментарий`
