# Режим: РЕРАЙТ (переписать целиком без привязки к изданию)

Полная переделка текста под стилевую систему — без контекста конкретного издания. Используется, когда нужно усилить текст в принципе, а адаптацию под издание Соня делает потом отдельно (или будет делать в режиме адаптации).

## Когда вызывается

- Соня даёт текст и просит «перепиши» / «сделай рерайт» / «переделай»
- Издание не указано или не важно для задачи
- Текст слишком слабый для редактуры — нужна переделка

## Чем отличается от адаптации

| | Адаптация | Рерайт |
|---|---|---|
| Контекст издания | Обязателен | Не нужен |
| Ресёрч сайта издания | Делается | Нет |
| База в CRM-таблице | Подтягивается | Нет |
| ЦА | Уточняется у Сони | По умолчанию широкая деловая аудитория, можно уточнить |
| Брендовый блок + анонс | Обязательно | Обязательно |

## Алгоритм работы

### Шаг 1. Короткая диагностика (5-7 предложений)

1. Где текущий текст теряет читателя
2. Какие 3-5 проблем нужно исправить в первую очередь
3. Какой тип конфликта использовать в новом лиде
4. Что из исходного текста сохранить
5. Что сократить или вынести

### Шаг 2. Переписать по 8 принципам

См. `editorial-criteria.md`. Применить все 8 принципов: конфликт в лиде, одна мысль на блок, бытовая расшифровка, без избыточной логики, чередование плотности, минимум 4 крючка из 7, не концепт-записка, простота без потерь.

### Шаг 3. Ограничения

- Объём итоговой статьи: 4000–7000 знаков (если Соня не указала иначе)
- Стиль — деловое медиа
- Тон живой, но не блоговый
- Подача журнально-аналитическая
- Без ИИ-пафоса, канцелярита, рекламной подачи
- Без лишних англицизмов при наличии русского аналога
- Зерокодер встраивается через опыт/исследования/кейсы, не как рекламный блок
- Каждый крупный тезис раскрывается через пример/факт/вывод
- Финал — польза: советы, чеклист, первые шаги, промпты

### Шаг 4. Формат ответа

```
1. Короткая редакционная диагностика
   [5-7 предложений]

2. Новый заголовок
   - Основной вариант
   - 3 альтернативы

3. Новая система подзаголовков
   [живые, содержательные, цепляющие]

4. Улучшенная версия статьи
   [полный текст]

5. Что изменено и почему
   - Какой тип конфликта использован в лиде
   - Где добавлен человеческий конфликт
   - Где усилен региональный мост
   - Где убрана лишняя логика
   - Где добавлены примеры
   - Где усилен ритм и какие крючки добавлены
   - Где Зерокодер встроен нативнее
   - Как усилен финал

6. Что проверить вручную
   - Где нужны ссылки на источники
   - Какие цифры требуют проверки
   - Какие формулировки могут быть спорными
   - Какие утверждения нельзя оставлять без подтверждения
   - Какие англицизмы заменены или требуют пояснения
```

## Главная цель

Не сделать текст просто «гладким». Сделать его **сильнее, яснее, быстрее, живее и читабельнее без потери смысла**.
