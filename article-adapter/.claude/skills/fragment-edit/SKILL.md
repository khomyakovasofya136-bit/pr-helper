# Skill: Правка фрагмента

Точечная микро-работа: маркеры антипаттернов → минимальные правки → пояснение каждой замены. Использует промт `articles/promts/mode-fragment.md`.

## Триггеры вызова

- «Поправь этот абзац»
- «Усиль лид»
- «Переделай заголовок»
- «Что не так с этим фрагментом»
- «Прокачай этот блок»

## Формат ответа

1. **Исходный фрагмент с маркерами** — пометить `[нейрослоп]`, `[не... а]`, `[перегруз]`, `[канцелярит]`, `[пафос]`
2. **Исправленный фрагмент** — минимальные точечные правки
3. **Пояснения к правкам** — для каждой замены: было → стало + причина
4. **Что не трогали и почему** — сильные места, серийные отсылки

## Что обязательно прочитать

- `articles/promts/mode-fragment.md`
- `articles/promts/style-reference.md`
- `articles/promts/editorial-criteria.md`

## Что НЕ делать

- Не возвращать полный текст статьи — только фрагмент (если Соня не попросила полный)
- Не переписывать смысл, только формулировки
- Не убирать брендовый блок + анонс, если он в фрагменте
- Не трогать серийные отсылки

## Главное

Соня должна видеть **почему** каждое изменение лучше — это образовательная функция режима.
