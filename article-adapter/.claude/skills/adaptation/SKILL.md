# Skill: Адаптация статьи под издание

Главный рабочий режим адаптера. Использует промт `articles/promts/mode-adaptation.md` — там полный алгоритм.

## Триггеры вызова

- «Адаптируй [статью] под [издание]»
- «Сделай версию [статьи] для [издание]»
- «Переделай под [издание]»
- «Сделай вариант для [VN / Т-Бизнес / HRlogia / другое]»

## Краткий алгоритм

1. **Собрать контекст**: shared/zerocoder-context, promts (style + criteria + brand-block), Google Sheets по изданию, publications/<изд>/rules.md, последний research, 1-2 examples
2. **Проверить свежесть ресёрча**: <30 дней — молча; 30-90 — спросить; >90 — сделать свежий через WebFetch/WebSearch
3. **Уточнить у Сони** 2 вопроса (см. mode-adaptation.md шаг 3):
   - Параметры из базы — актуальны?
   - ЦА: типовая подходит или другую?
4. **Адаптировать текст** по всем правилам
5. **Выдать** в формате: статья + список изменений с обоснованием + что проверить вручную
6. **Спросить** про обновление CRM-таблицы

## Куда сохранять

`articles/outputs/[ГГГГ-ММ]_[издание]_[тема]_adapted.md`

## Что обязательно прочитать

- `articles/promts/mode-adaptation.md` — полный алгоритм
- `articles/promts/style-reference.md`
- `articles/promts/editorial-criteria.md`
- `articles/promts/brand-block-pattern.md`
- `pr-helper/shared/zerocoder-context.md`
- `articles/publications/<издание>/rules.md`
- `articles/publications/audience-types.md`

## Что НЕ делать

- Не писать статьи с нуля
- Не предполагать ЦА молча — всегда спрашивать
- Не использовать ресёрч старше 90 дней без свежения
- Не обновлять CRM без подтверждения Сони
- Не удалять брендовый блок + анонс
