# Topic Discovery — поиск тем для b2b-чата

Сценарий B из описания агента: **«анализ медиаполя + фактуры Зерокодера → рекомендации по теме»**. Финальное написание поста делает скилл `AI-News to Social` после выбора Соней.

## Когда вызывается

- «Дай темы для b2b-чата»
- «Что написать в чат на этой неделе»
- «Посмотри, о чём можно написать»
- «Подбери тему для поста»

**Запуск — только по запросу Сони.** Не по расписанию, не автоматически.

## Что подгружает

1. `pr-helper/shared/zerocoder-context.md` — бренд, риск-зоны, что нельзя
2. `pr-helper/b2bchat/references/top-posts.md` — что РАБОТАЕТ для b2b-аудитории (инсайты + примеры)
3. **Свежий дайджест newsjacker** — последний файл из `pr-helper/newsjacking/YYYY-MM-DD_digest.md`
4. **Канал Кирилла Пшинника** — `https://t.me/s/k_pshinnik` (через WebFetch) — последние посты
5. **Блог Зерокодера** — `https://ya.zerocoder.ru/` (главная) и `https://ya.zerocoder.ru/tag/cases/` (кейсы)
6. **Все публикации Зерокодера** — `https://zerocoder.ru/news` (новостная лента сайта)

Дополнительно при необходимости (из `shared/zerocoder-context.md`):
- Публикации в Т-Бизнес Секретах, РБК, Forbes, CNews
- Обучение госсектора
- Материалы «Я — Зерокодер»

## Алгоритм работы

### Шаг 1. Сканирование

Прочитать в указанном порядке (можно параллельно):

```
1) Последний дайджест newsjacker (за последние 1-3 дня)
2) Канал Кирилла Пшинника — t.me/s/k_pshinnik
3) Блог Зерокодера — ya.zerocoder.ru (свежие материалы)
4) Все публикации Зерокодера — zerocoder.ru/news (свежие)
```

Если newsjacker-дайджеста за сегодня нет — взять самый свежий доступный.

### Шаг 2. Отбор кандидатов

Фильтр по критериям успешного поста (см. `references/top-posts.md`):
- ✅ Можно подать **через внутренний кейс Зерокодера** с цифрами — приоритет 1
- ✅ Это **исследование с неочевидным выводом** — приоритет 2
- ✅ **Серийность с уже зашедшей темой** (вайб-кодинг, ИИ-агенты) — приоритет 3
- ❌ Просто новостной пересказ без своего угла — **отсеять**

### Шаг 3. Формирование 3-5 рекомендаций

Для каждой темы выдать:

```markdown
### N. [Заголовок темы — конкретный, с углом]

**Источник:** [откуда — newsjacker / Кирилл / блог / комбинация]
**Тип:** внутренний кейс / исследование / разбор тренда / репортаж
**Релевантность:** 8/10

**О чём:** 2-3 предложения — суть инфоповода или фактуры.

**Почему зайдёт b2b-аудитории:** 2-3 предложения — связь с эталонами, аналогия с топ-постом если есть.

**Угол подачи:** конкретный тезис, который будет в посте. Не «расскажем про X», а «X работает только если Y».

**Что нужно для написания:** список (цифры, кейсы, спикер, дополнительные источники).
```

### Шаг 4. Передача дальше

В конце сообщения:

> Выбери одну тему — и я зову `AI-News to Social` для написания поста.
> Или скажи, что доработать в рекомендациях.

После выбора Соней — пишет пост через скилл `AI-News to Social` (тот же стиль, та же структура).

## Что НЕ делать

- **Не писать пост сразу** — только рекомендации. Пост = `AI-News to Social` после выбора.
- **Не предлагать новостные пересказы без угла** — это анти-паттерн (см. analytics).
- **Не предлагать более 5 тем** — лишний шум, Соне выбирать тяжелее.
- **Не повторять темы, которые уже выходили недавно** — проверить `output/` за последние 2-4 недели.
- **Не использовать данные без источника** — если в дайджесте или фактуре нет цифры → пометка `[нужен источник]`.
